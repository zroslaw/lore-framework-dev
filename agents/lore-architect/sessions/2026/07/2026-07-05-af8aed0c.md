# Built and validated a Codex-ready port of the lore framework

The session opened as a focused push on the multi-engine-portability direction, narrowed to Codex. The prior state was that only the mechanical framework-root change set had been validated (on Claude/haiku) and staged in an isolated copy; the hard Tier-B "subagent-spawn" nucleus and the `docs/engines/` adapter layer were still design, not build. The user chose to work empirically — run the framework on real Codex and fix what actually broke — rather than reason about it on paper.

Two initial Codex trials (`codex exec`, `gpt-5.4-mini`) surfaced the expected frictions: empty `${CLAUDE_PLUGIN_ROOT}`, slash-skills falling through to the shell, and a sandbox-blocked `ps` teammate probe. A full finalize ran faithfully but merge fell back to inline "because no subagent path was exposed." That last conclusion turned out to be the session's central plot twist. A research subagent, cross-checked against the installed binary, established that Codex 0.142.5 ships a first-class native multi-agent subsystem — feature `multi_agent` stable and on by default, with `spawn_agent`/`wait_agent`/`send_message` tools, worker/explorer roles, ad-hoc spawning, addressed inter-agent messaging, and a depth-1 team model. The earlier "no native subagents" read was wrong; merge had gone inline only because nothing instructed the model to spawn.

With capabilities understood, the work turned to design and then implementation. The `docs/engines/` convention was specified — one profile per engine filling the same five bindings (framework-root, invocation-syntax, subagent-spawn, memory-file, runtime-bounding), selected by a new Boot Step-0 that self-locates the framework root, infers the engine, and lets the profile win on conflict. A key sub-decision: keep the shared procedure docs engine-agnostic and put the Codex-specific handling in the profile, including a fan-out override where the host reads the merge/recall procedure and passes the steps inline to each subagent (honoring Codex's "the main agent reads the instructions" convention) while the subagent still boots as its target agent. The whole binding was then built into an isolated copy, `lore-framework-codex`: a mechanical framework-root pass over 44 files plus self-location lines in 27 SKILL.md, the two engine profiles, Boot Step-0, and one-line engine-note pointers at the four fan-out sites.

The ported build was then validated on real Codex, against throwaway repo copies. Boot selected the codex profile, framework-root self-located with zero token leak, and — the decisive result — both recall and merge ran through native `spawn_agent` (explorer for recall, worker for merge) with the host-reads-steps override, confirmed not by the model's self-report but by inspecting Codex's own session rollout logs, which recorded the real `function_call` items in the `multi_agent_v1` namespace. Merge integrated its reflections into lore and cleaned up correctly. The single remaining gap was environmental, not a design fault: Codex's sandbox blocks writes under `.git/`, so `git commit` (and auto-pull) cannot run under the default `workspace-write` policy — captured as an operational note with the fix (full-access or user-commits-by-hand).

The session landed with the Codex port no longer a plan but a working, end-to-end-validated build sitting beside the live plugin, which stayed untouched throughout. The two real findings from an earlier throwaway trial were ported into the real repo, the throwaway copies were cleaned up, and this finalization folded the session's knowledge into lore: five new topics (the engine-profile convention, the native subsystem, the validation result, the git-sandbox gap, the testing methodology) and updates across the portability direction, the next-steps landing plan, and lore-context.

Next steps are to fold `docs/engines/` into the canonical `lore-framework` (with `CLAUDE.md`→`AGENTS.md` and `test_18_init` in lockstep), convert the still-deferred token sites (`.mcp.json`/lr-wait, `migrations/*`, `df/aiqa`), decide the git-sandbox handling for Codex commits, test interactive-Codex skill dispatch (only headless was exercised), and wire an automated `codex` driver into the lifecycle harness so this stops being a manual loop.
