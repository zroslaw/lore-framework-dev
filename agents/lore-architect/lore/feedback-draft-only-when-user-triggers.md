# Feedback — In Design Dialogues, Write the Draft Only When the User Says So

Corrective feedback from the user during the 2026-07-19 Lore Beings design dialogue.

## What happened

Twice during the dialogue I moved to fold agreed decisions into the draft file mid-conversation. The user interrupted: "We're not done yet. Why are you always trying to put these things in a draft? When we will be ready, I will tell you."

## Why it was wrong

In a dialogue-mode design session, the conversation itself is the working surface — decisions are safe in context. Premature persistence interrupts the flow, signals I think the discussion is over, and creates stale intermediate artifacts that need rewriting anyway.

## How to apply

- During design dialogues, accumulate decisions in conversation and keep confirming them verbally.
- Touch the draft/artifact only when the user explicitly triggers it ("let's have this draft").
- Offering once at a natural milestone is fine; repeatedly moving to write is not.
