# Workspace layer design for v25

## What happened

User booted lore-architect in Cursor to explore **workspace-as-git-repo** and multi-repo layout for
`agent-workspace`. Evolved into a full v25 design slice extending cursor-ops-parity:

- **`/lr:workspace-pull`** (hard rename from workspace-sync) — two-level pull: workspace
  `lore-workspace.md` repos, then domain `lore-repo.md` `repos:`; optional workspace-root git pull;
  gitignore plumbing for declared children.
- **`/lr:workspace-init`** (hard rename from init) — `git init`-like setup wizard + AGENTS.md
  payload v2; no separate create-workspace skill.

Ran **three-lens review cycle** (architecture, UX, implementation) over three rounds; specs
converged to **approve to implement**. Drafts kept in `workdir/`; UX review in
`review-v25-workspace-ux.md`.
