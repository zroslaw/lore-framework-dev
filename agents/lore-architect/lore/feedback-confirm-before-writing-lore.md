# Feedback — "Take a Note" ≠ Authorization to Write Lore

Corrective feedback from the user this session (2026-06-03).

## What happened

When the user said a finding was "interesting" and later "please take a note about the case above," I immediately wrote a full lore topic and edited `lore-context.md`. The user pushed back twice: "what did you do? did I ask you to do anything?" and then "revert … let's first talk about it." Both edits were reverted.

## Why it was wrong

"Take a note" / "this is interesting" is a signal to *register and discuss* something, not a license to commit durable lore mid-session. Writing lore is a finalization-time act (reflect → merge), gated and reviewable. Doing it inline, unprompted, both jumps the gun and picks a format/scope the user never agreed to.

## How to apply

- In a **design-exploration session, default to discussion**: reflect the idea back, confirm scope, and let the user drive what (if anything) gets written. Persist only at explicit finalization or on a clear "write this down as lore."
- This is the **follow-me discipline** generalized: small suggestions, don't race ahead, don't overtake the user's thinking. The user explicitly turned on follow-me mode right after this and the session went notably better ("I like how our conversation has been going"). Treat follow-me as the *default posture* for open-ended design work, not just when asked.
