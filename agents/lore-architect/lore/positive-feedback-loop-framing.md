# Positive feedback loop between usage and learning

Name this dynamic explicitly when explaining why Lore Agents are self-sustaining. The headline phrase **"a positive feedback loop between usage and learning"** is the right register — it's a recognizable technical concept that pre-encodes the self-reinforcement claim, and it's compact enough to use as a section opener.

The loop:

1. People reach for Lore Agents not out of obligation to share knowledge, but because they're the **most capable agents in their specific areas** — with the largest and most recent context.
2. Daily execution captures new context, refines procedures, surfaces operational lessons.
3. The agent becomes more capable.
4. The next user reaches for it for the same reason — practical value — and adds more.

The loop is self-sustaining because **the user's incentive (getting their own work done faster) is exactly what feeds the agent's growth.** Misalignment between user incentive and KB-curation burden is what breaks every traditional knowledge base; this framework is structured so the two are the same activity.

Companion: skill-set growth is the most explicit place this loop runs. **"Skill set growth made easy and frictionless, embedded in the work itself"** captures the why — when improvements are easy to make (in-session, no separate maintenance repo, no "skill authoring mode" context-switch), they happen often, and that frequency feeds the loop.
