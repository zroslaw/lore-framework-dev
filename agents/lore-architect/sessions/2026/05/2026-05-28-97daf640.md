# Expanded Activities LORE_AGENTS_INTRO with Anatomy, Domain, agents, and use cases

The session opened with the user picking up the in-flight Activities team onboarding doc. The v1 of `LORE_AGENTS_INTRO.md` had landed §1 ("The idea") and §4 ("Features") earlier; several middle sections were still placeholder stubs. The user wanted help drafting §3 "Anatomy", a Domain Concept section, §2 "List Of Activities Lore Agents", §2 "Some Use Cases (so far)", and a meta acknowledgement of the present session itself.

The Anatomy section was composed against the four-piece agent shape (`role.md`, `lore-context.md`, `lore/`, `workdir/`) with three blockquote beats — "A role, a context, a body of lore, and a workdir" / "Boot → work → finalize" / "Plain markdown, in Git, by design" — plus a compact `tree`-style directory visual the user requested partway through to give readers a concrete sense of what the layout looks like on disk. The Domain Concept section was kept tight: a single headline, a paragraph defining the LRF "Domain" term against the Activities example (activity-* repos plus MPBE/BCRE, MSPA, protobufs), and two consequences (free discovery, lore stays focused on the operational layer rather than duplicating code). The agents list was assembled by reading every `role.md` under `activities-lore-agents/agents/` — 22 agents grouped by purpose into Domain & cross-cutting / Repository experts / Supplier integrations / Project & RFC trackers / Team & process. The user pushed back on the initial framing because it suggested agents only mirror components; the section was extended with an explicit "Agents come in many shapes" beat naming the four shapes that already exist in the Activities domain (components, projects, practices/tooling, roles/team-ops).

The use-cases section was the showcase. Five `/lr:consult` calls dispatched in parallel and in the background to `activity-supply-manager`, `oncall-agent`, `globaltix-api`, `activity-supply`, and `custom-pax-types`. Each consultant booted in its own subagent, searched its own lore, and returned a structured synthesis with concrete artifacts — Slack permalinks, ABE ticket IDs, capacity numbers (Q2 192 SP / Q3 2.3× overcommit), workdir scripts (`verify_empty_calendars_prod.py`), top-20 supplier_code↔activity_id mappings, and RFC sequencing across Q2/Q3/Q4. Returns were woven into seven subsections, each tagged with the contributing agents. The user asked to add a closing meta subsection naming the present session and the consults used — the doc thereby becomes a literal demonstration of the loop §1 asserts.

A plot twist surfaced during consults: `oncall-agent` flagged that the user's framing of one use case ("KKday allowed origins") didn't match what the consultant's lore actually held — the relevant investigation had been an empty-offers / pax-shape mismatch, not an origin-whitelist question. The doc's framing was adjusted to acknowledge both readings rather than forcing the lore to fit the original framing. Numbering of sections drifted as content was added — a polish pass renumbered to 1–8 (idea, anatomy, domain, agents list, use cases, iTerm2 & Agent Teams TBD, Swarm Of Agents TBD, features). The user briefly worried the two TBD placeholders had been dropped during the renumber; they had been, briefly, and were restored in a follow-up commit. A small `iTerms2`→`iTerm2` typo was corrected in the same pass.

Operational note: GitHub authentication on this machine surfaced as a small detour. The local `~/.ssh/github_ed25519` already authenticates as `zroslaw`, but both `lore-framework-dev` and `lore-framework` had HTTPS remotes that would have prompted for credentials on push. Both remotes were rewritten to `git@github.com:zroslaw/...` so the existing SSH identity is used directly.

The session landed with two GitLab pushes to `activities-lore-agents` (commits `56b8274` and `f24b318`) — both forms of the polished intro doc are now on `origin/main`. Three reflections were extracted and merged into lore: the existing `onboarding-doc-narrative-pattern.md` topic was extended with the §3 Anatomy and §2 meta-subsection guidance; a new standalone topic `use-cases-via-parallel-consult-pattern.md` was created for the parallel-consult fan-out procedure; `lore-context.md`'s Onboarding Doc Authoring section was updated to reference both. The §6 (iTerm2 & Agent Teams) and §7 (Swarm Of Agents) sections of the intro remain TBD — open threads for a future session.
