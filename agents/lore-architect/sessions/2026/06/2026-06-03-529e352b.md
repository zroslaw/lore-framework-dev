# Evolved AIQA per-file store into the per-repo codex

The session opened on a practical errand that turned into a design step. The user wanted to exercise the new dev ULA (unit-level analysis) BETA features against a real codebase, and pointed at `My-Turbo-Boost-Switcher` — a macOS menu-bar Objective-C app that toggles Intel Turbo Boost through a kernel extension. After surveying the source files, `SystemCommands.m` was chosen as the most behaviorally rich target (privileged kext load/unload, SMC sensor reads, OS/arch branching), and from it the five most important units were ranked. The user then narrowed the test to a single unit, `run-task-as-admin`.

That single-unit ULA pass proved the feature's worth on its own. It surfaced six potential bugs, the most important being `always-returns-yes` — the method logs authorization failures but unconditionally returns `YES`, the root of a three-layer error-swallowing chain (`runTaskAsAdmin` → `loadModuleWithPath` → `AppDelegate`, each ignoring the one below) — and `deprecated-api`, the fact that `AuthorizationExecuteWithPrivileges` is gone on macOS 14+, silently breaking the app's core mechanism on current hardware. Getting there required two workflow reruns: the dynamic Workflow `args` payload arrived as a string and broke destructuring, and the fix was to abandon args injection for a self-contained script with inputs hardcoded; single-unit scoping was achieved by bypassing the splitter and feeding the explicit unit straight into the Unit Pass. Artifacts (`bugs.yaml`, `scenarios.yaml`, `gap.yaml`) were persisted into a freshly scaffolded `My-Turbo-Boost-Switcher-aiqa` sibling repo.

The conversation around those bugs is what opened the design thread. Interpreting the findings produced a kind of knowledge — the error-swallowing chain, the deprecation story, the severity calibration that the fd-leak is near-zero impact for a menu-bar app while the quiet deprecation is the real killer — that fits none of the three ULA schemas. The user recognized this as the long-discussed "file lore" / "context per file" need, and a recall confirmed prior design work: the context-agent + per-file-mirror idea in `lr-dev-direction.md`, `quality-repo-architecture.md`, and `draft-lr-dev-quality.md`.

From there the user drove a consolidation. They considered specialized per-repo agents (a quality agent and a context agent, two aspects of one repo) but talked themselves out of agents entirely: if persistence is an external shared repo and the procedures live in skills, an agent has no function or lore of its own. The landing point: the artifact store generalizes from a quality-only `-aiqa` repo into a broader **codex** (`<repo>-codex`) that mirrors the source tree; each source file gets a directory created lazily — only once something is worth preserving, so the filesystem itself becomes a sparse coverage map; a per-file `index.md` holds the narrative ("context") aspect while structured disciplines hang off as subdirs (`aiqa/` today); skills are aspect-scoped and standardized at the lr-dev/framework level, with the per-repo "instance" being just the codex repo plus a small config rather than an agent. Names were chosen deliberately: `codex` (over atlas/mirror/shadow) and `index.md` (over overview/summary/dossier).

There were two corrections worth recording. Early on, when the user said "take a note," the host immediately wrote a lore topic and edited `lore-context.md`; the user pushed back ("did I ask you to do anything?") and had both reverted — "take a note" is a signal to register and discuss, not to commit durable lore mid-session. The user then explicitly enabled follow-me mode, and the rest of the session — reflecting ideas back, offering small suggestions, not racing ahead — went notably better by their own assessment. That working-style lesson was captured as feedback lore and folded into the follow-me soft-skill topic as the default posture for open-ended design work.

The session landed with the codex direction merged into lore: four new topics (led by `codex-per-repo-mirror.md`) and seven updated, with the superseded context-agent and three-repo framings demoted via banners rather than deleted so the reasoning trail survives. Deferred threads: renaming/rescoping `dev-aiqa-repo-init` into an aspect-agnostic codex-init skill, migrating the existing `My-Turbo-Boost-Switcher-aiqa` repo into the `…-codex/SystemCommands.m/aiqa/…` + `index.md` shape as a worked example, writing the codex layout/vocabulary down as the canonical lr-dev spec, and the parked question of repo-level (codex-wide) topics above the per-file layer.
