# The agent as universal working environment

A framing that lands well when explaining where Lore Agents fit in a developer's daily life. **People increasingly do *everything* through an agent** — writing code, reviewing MRs, arranging tickets, querying logs, reading runbooks, drafting messages.

The agent is no longer a tool you summon for a specific question. It's the **unified interface to everything around the work**:

- Keeps and tracks context as the user moves between tasks.
- Integrates with the systems they use (git, JIRA, Confluence, Grafana, Slack, ...).
- Remembers where they left off.

A **Lore Agent** extends this from a *generic* working environment into a *specialized* one — pre-loaded with the area's knowledge and skills. Not just a UI to your tools, but a colleague that already knows the area.

This framing is useful because:

1. It positions Lore Agents inside an emerging behavior pattern (working-through-the-agent) rather than asking the reader to imagine a new behavior.
2. It explains *why* the accumulated knowledge matters — context-tracking and integration are general-purpose; *area-specific knowledge* is what makes a Lore Agent more capable than a generic agent in that area.
3. It's a natural bridge from §1 (the idea) to §2 (use cases) and §3 (anatomy) of an onboarding doc — once the reader sees the agent as the working environment, the per-agent specialization is the obvious payoff.
