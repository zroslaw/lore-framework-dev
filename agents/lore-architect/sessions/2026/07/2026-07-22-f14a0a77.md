# Shipped v28 lifecycle gate

The session opened with a release-status audit for Lore framework v28 after the user asked whether the previous end-to-end testing work was enough to ship safely. Lore Architect booted against the current v28 framework, checked the `lore-framework` and `lore-framework-dev` repos, and found that v28 was prepared locally but not yet pushed or tagged.

The work first improved the test pipeline instead of rerunning expensive tests blindly. A new lifecycle matrix runner, `tests/lifecycle/run_matrix.py`, now runs standard or Keeper lifecycle suites across engines, stores durable JSON summaries and per-module logs, supports targeted reruns through `--test-filter`, and keeps raw result directories out of git. The test README and Lore Architect lifecycle-harness topic were updated to make this the normal release-gate path.

The real-engine v28 gate then ran with persisted evidence. The broad standard lifecycle matrix passed 16 of 18 engine/module jobs, with failures limited to Claude boot scenario 05 and Cursor repo/workspace scenario 21. Targeted reruns of exactly those two failed scenarios passed, so every standard lifecycle scenario had at least one passing v28-candidate run. The deterministic test suite also passed, and the gate evidence was recorded in `workdir/v28-e2e-gate-2026-07-22.md`.

The user pushed for short, readable status and asked for cost/timing visibility. The session reconstructed the known cost and duration from logs: about 58.5 minutes total for the v28 gate sequence, with known Claude spend around $3.02 and Codex/Cursor USD unavailable in current logs. A new standing backlog item now calls for structured metrics across unit, standard lifecycle, Keeper lifecycle, and quality benchmark runs, including wall time, per-test timings, engine/model metadata, result paths, and costs where engines expose them.

The session landed by shipping Lore framework v28: `origin/main` now points at `5e00209`, and tag `lr--v1.28.0` was pushed. Lore Architect's context and version-history topic were updated from "prepared locally" to "shipped", and the old v27 partial-gate follow-up was marked covered by the v28 gate.
