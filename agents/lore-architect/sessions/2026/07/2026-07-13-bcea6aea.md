# Shipped v25 and prepared marketplace submission

The session opened as a takeover of a Claude Code `lore-architect` session that had already booted
the agent, run the full lifecycle suite across Claude, Codex, and Cursor, and stopped while
investigating several failures. The first task was to continue from the digest rather than restart:
the session restored the lore-architect identity in Codex, verified on-disk state, and treated the
old Claude task list as stale until each item could be mapped back to actual logs, diffs, and git
state.

The central technical work was closing the v25 release. The full suite had exposed one real
framework clarity gap and several harness/model-variance issues. `docs/check.md` gained the same
current-working-directory guard that boot already carried, the lifecycle harness was updated for
`workspace-init` and `workspace-pull`, and debug affordances were added for retained fixtures and
final-message dumps. Targeted re-runs confirmed the repaired cases: haiku passed `check`,
`summarize`, and `create-agent` in isolation, while Codex `gpt-5.4-mini` passed
`workspace-init`. The user explicitly accepted shipping without a fresh full three-engine rerun
after those targeted confirmations.

Release hygiene then became the focus. The PNG logo migration was accepted: Cursor and Codex
manifests were repointed to PNG assets and the previous SVG logo was removed from the distributed
plugin tree. The meta workspace descriptor was refreshed for v25 and `.DS_Store` was ignored.
`lore-framework` was committed and pushed through the full v25 stack, `lore-framework-dev` received
the lifecycle harness, quality matrix, and lore updates, and the meta workspace received its local
descriptor refresh. The runtime release was tagged with `lr--v1.25.0` at commit `09fe4f0`.

The final plot twist came from marketplace submission. The Claude Console form asked for usage
examples and then for a privacy-policy URL, which revealed that manifest metadata alone was not
enough for public review. The repo gained a root `MARKETPLACE.md` with copy/paste submission
metadata for Claude Code, Codex, and Cursor, plus a root `PRIVACY.md` describing local storage,
workspace-scoped access, explicit git/network use, and absence of telemetry. These support files
were committed and pushed after the runtime release tag, so lore now distinguishes runtime release
identity from submission-support metadata.

The session landed with all working trees clean before finalization, Claude plugin validation still
strict-clean, `lore-framework` pushed through `72e7bda`, `lore-framework-dev` pushed through
`e6eb8fd`, and the meta workspace clean locally with no remote configured. During finalization,
the marketplace-submission lesson was merged into `engine-marketplace-readiness.md`,
`plugin-distribution.md`, and `lore-context.md`. The only unresolved external action is the actual
Claude Console submission, which remains gated on the user completing the authenticated form; Codex
and Cursor publication claims remain intentionally conservative until their live public marketplace
paths are verified.
